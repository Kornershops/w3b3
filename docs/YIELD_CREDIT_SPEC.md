# Yield-Backed Credit: Implementation Plan & Acceptance Criteria

## 1. Feature Overview
The **Yield-Backed Credit** module allows users to borrow stablecoins (or native assets) against their active staking positions ($W3B3-LPs) without needing to unstake. This maximizes capital efficiency, similar to Aave or MakerDAO, but specifically utilizing yield-bearing assets as collateral.

## 2. Acceptance Criteria (AC)
- **AC1: Collateralization**: Users must be able to lock their yield-bearing tokens (e.g., staked ETH receipts or W3B3 LP tokens) as collateral.
- **AC2: Borrowing**: Users can borrow stablecoins up to a defined Loan-to-Value (LTV) ratio (e.g., 50% of the collateral's USD value).
- **AC3: Interest Accrual**: The loan should accrue interest over time at a dynamic or fixed borrow rate.
- **AC4: Yield Offset**: The yield generated by the locked collateral should automatically offset the borrow interest.
- **AC5: Liquidation**: If the collateral value drops below the liquidation threshold, liquidators can repay the debt in exchange for a portion of the collateral (plus a bonus).
- **AC6: Dashboard (UI)**: The frontend should display a user's "Health Factor," borrowed amount, available credit, and net APY (Yield APY - Borrow APY).

## 3. Architecture components
1. **Smart Contracts**: `W3B3CreditLine.sol` (handles deposits, borrowing, liquidation, and interest math).
2. **Backend**: Index credit positions, health factors, and liquidation opportunities.
3. **Frontend**: The "Borrow" tab in the portfolio view.

## 4. Test Strategy
### Smart Contract Tests (Hardhat/Chai)
- `test_borrowing_under_ltv`: Ensure users cannot borrow more than their LTV.
- `test_interest_accrual`: Fast-forward time and verify interest accumulates correctly.
- `test_liquidation`: Simulate a price drop and successfully liquidate a position.
- `test_yield_offset`: Verify that collateral yield reduces the effective debt.

### Backend Tests (Vitest)
- Mock on-chain reads and verify Health Factor calculations match the contract.

### Frontend Tests (Vitest/React Testing Library)
- Test the borrowing form inputs and health factor warnings.

## 5. Next Steps for Implementation

2. Write unit tests for the contract.
3. Integrate real-time price feeds for LTV calculation.
